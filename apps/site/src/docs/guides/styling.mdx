## CSS variables

Override `--ezlet-*` tokens for theme-level changes. These affect all toasts globally.

```css
:root {
  --ezlet-bg: #1a1a1a;
  --ezlet-color: #ffffff;
  --ezlet-border-radius: 14px;
  --ezlet-border: 1px solid rgba(255, 255, 255, 0.08);
  --ezlet-shadow: 0 8px 32px rgba(0, 0, 0, 0.4);
}
```

## classNames

Target named slots — `toast`, `title`, `description`, `action`, `icon`:

```tsx
<Toaster
  classNames={{
    toast: "border border-zinc-800 bg-zinc-900",
    title: "font-semibold text-white",
    description: "text-zinc-400",
    action: "bg-zinc-700 hover:bg-zinc-600 text-white",
  }}
/>
```

## Fully custom toasts

Use `toast.custom()` when one notification needs its own JSX and styling while keeping Ezlet's
stacking, timers, dismissal, and motion:

```tsx
toast.custom((item) => (
  <div className="flex w-full items-center rounded-lg bg-white p-4 shadow-lg ring-1 ring-black/5">
    <div className="min-w-0 flex-1">
      <p className="text-sm font-medium text-gray-900">Headless toast</p>
      <p className="mt-1 text-sm text-gray-500">Full control of markup and styles.</p>
    </div>
    <button
      className="ml-4 rounded bg-indigo-50 px-3 py-1 text-sm font-semibold text-indigo-600"
      onClick={() => toast.dismiss(item.id)}
      type="button"
    >
      Close
    </button>
  </div>
));
```
